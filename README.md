# marketing-budget-decision

마케팅 성과 CSV 한 장을 넣으면 "다음 주 예산을 어느 채널로 옮길지"를 금액까지 답하는 리포트를 자동 생성한다.
계산은 Python(pandas)이 전담하고, 해석과 권고만 Claude가 담당한다.

```bash
python scripts/generate_insight_report.py --csv data/marketing_performance.csv
# → insight_report.md + metrics.json 재생성
```

| 채널 | 광고비 비중 | 매출 비중 | ROI | 권고 |
|------|-----------:|---------:|----:|------|
| 이메일 | 38.5% | 46.3% | 1,042% | 유지 + 초과분 수용 |
| 네이버 | 22.5% | 17.1% | 659% | 유지·소폭 증액 |
| **메타** | **26.0%** | **7.8%** | **259%** | **일 예산 한도·이동** |
| 카카오 | 13.0% | 5.1% | 343% | W4 급락 원인 확인 |

메타 단일 채널이 유료 예산의 1/4을 쓰면서 매출은 1/13만 만든다. 예산이 효율과 반대로 배분된 상태.

전체 근거와 반론 대응은 [`의사결정메모.md`](의사결정메모.md), 상세 분석은 [`insight_report.md`](insight_report.md) 참조.

## 이 리포트를 읽는 사람과, 왜 필요한가

**독자: 주간 채널 예산을 직접 배분하는 퍼포먼스 마케팅 리드.**

이 담당자는 매주 같은 결정을 반복한다. 어느 채널을 늘리고, 어디를 줄이고, 어디는 손대지 않을지. 결정의 근거는 5개 채널 성과 CSV 한 장뿐인데, 정작 그 안에는 "그래서 어디로 옮기나"라는 답이 정리돼 있지 않다.

| 담당자의 현실 | 이 리포트가 바꾸는 것 |
|---------------|----------------------|
| 매주 수작업 추출·계산·요약에 반나절 소요 | 명령 1회, 1분 이내 재생성 |
| 엑셀 수식 계산은 오류가 나도 검증 불가 | 전 계산 pandas 전담 + 재검증 |
| 결과물이 수치 나열이라 결정으로 미연결 | 채널 순위·이슈·이동 금액까지 제시 |
| 효율 낮은 채널로 예산이 새는 것을 사후 인지 | 급변 이슈를 배포 전 자동 탐지 |

즉 이 리포지토리의 목표는 리포트 자동 생성기가 아니라, **담당자가 추가 분석 없이 그대로 예산을 옮길 수 있는 결정 도구**다.

## 분석 설계: 문제 정의 → 가설 → 검증 → 결과

담당자의 질문 하나("다음 주 어느 채널로 옮기나")를, 답에 필요한 조건 네 개로 분해해 가설을 세우고 순서대로 검증했다. 각 가설은 "이게 성립해야 예산 이동이 정당화된다"는 논리로 연결된다.

### 가설 1 — 채널 간 효율 격차가 이동을 정당화할 만큼 큰가

- **문제 정의**: 격차가 작으면 어느 채널에 넣든 결과가 비슷해 이동할 이유가 없다. 이동이 성립하려면 격차부터 커야 한다.
- **가설**: 채널 간 ROI 격차가 예산 재배분을 정당화할 만큼 크다.
- **검증**: 채널별 ROI 산출. 이메일 1,042% vs 메타 259%로 약 4배. 나아가 메타는 유료 광고비의 26.0%를 쓰면서 매출은 7.8%만 기여, 이메일은 38.5%를 써서 46.3%를 기여.
- **결과**: **채택.** 격차 자체보다 예산이 효율과 반대로 배분된 점이 문제. 이동의 1차 근거 확보.
- **담당자에게 왜 중요한가**: 같은 금액을 채널만 바꿔 결과를 몇 배로 벌릴 여지가 있다는 뜻. 증액이 아니라 재배분만으로 성과를 올릴 수 있다.

### 가설 2 — 그 격차가 '잘한 것'인지 업계 기준으로 확인되는가

- **문제 정의**: 내부 순위만 보면 "이메일 1위"에서 끝난다. 1위라 더 넣으면 되는지 판단할 절대 기준이 없다.
- **가설**: 업계 벤치마크와 비교하면 내부 순위와 다른 그림이 나온다.
- **검증**: 업계 기준 대비 판정. 네이버 ROAS 6.59는 업계 상위 25% 진입. 이메일 10.42는 내부 1위이나 업계 평균(18.5) 하회.
- **결과**: **부분 반증.** 이메일은 내부 최고이나 업계 대비 여유가 남음. 결론이 "이메일 무조건 증액"에서 "증액 전 발송 최적화 우선"으로 수정됨.
- **담당자에게 왜 중요한가**: 1위 채널에 예산을 더 부어도 업계 평균에 못 미칠 수 있다. 증액 전 개선 여지를 먼저 점검해야 낭비를 막는다.

### 가설 3 — 최근 예산이 새는 구간이 실제로 있는가

- **문제 정의**: 8주 평균만 보면 괜찮아 보여도 특정 주에 사고가 났을 수 있다. 평균은 사고를 감춘다.
- **가설**: 주차·채널별로 분해하면 전주 대비 ±50%를 넘는 급변이 잡힌다.
- **검증**: 주차별 변화율 산출. 메타 W4→W5 광고비 +255.9%인데 매출 +43.6%에 그쳐 ROI 306%→123% 반감. 초과 집행분 319만원은 그 주 유료 예산의 35.2%. 카카오는 W3→W4 광고비 -41.2%이나 ROI는 정상 = 집행 자체 누락.
- **결과**: **채택.** 메타는 과집행, 카카오는 집행 누락. 이동할 재원 319만원의 출처가 여기서 특정됨.
- **담당자에게 왜 중요한가**: "어디서 빼서 어디로 옮기나"의 출처가 명확해진다. 막연한 증감이 아니라 특정 낭비분을 지목해 이동할 수 있다.

### 가설 4 — 이 숫자를 신뢰해도 되는가

- **문제 정의**: 앞의 결론이 전부 데이터가 정확하다는 전제 위에 있다. 데이터가 틀리면 결론도 무너진다.
- **가설**: 원본을 그대로 쓰면 이상치·중복이 순위를 왜곡한다.
- **검증**: 이메일 04-26 매출 1건의 AOV가 평균의 9.6배. 포함 시 이메일 순위 과대. 완전 중복 1행, 결측 3건도 확인.
- **결과**: **채택.** 중복 제거 → 결측 0 대체 금지 → 이상치 제외 순으로 정제 후 집계. 정제 없이는 가설 1~3의 순위가 뒤집힘.
- **담당자에게 왜 중요한가**: 잘못된 한 줄이 채널 순위를 통째로 바꾼다. 정제 규칙을 고정해야 매주 같은 신뢰도의 결정을 내릴 수 있다.

> 네 가설이 모두 성립하면 하나의 액션으로 수렴한다: **메타 초과분 319만원 → 이메일 이동.** 가설별 수치 근거는 `insight_report.md` §3~5, 정제 판단은 [`decisions.md`](decisions.md) 참조.

## 데이터 처리 규칙

결론의 신뢰성은 정제에서 나온다(가설 4). 원본 280행에 다음을 적용했다.

| 처리 | 대상 | 이유 |
|------|------|------|
| 중복 제거 | 네이버 05-15 1행 | 유지 시 성과 과대 집계 |
| 결측 유지(0 대체 금지) | 3건 | 수집 누락을 0 성과로 처리 시 합계 왜곡 |
| 이상치 제외 | 이메일 04-26 (AOV 9.6배) | 포함 시 이메일 순위 왜곡. 삭제가 아닌 제외 후 명시 |
| ROI 측정 불가 처리 | 오가닉(광고비 0) | 0·무한대 표기는 오답 |

280행 → 중복 제거 279 → 이상치 제외 278로 집계. 모든 합계·ROI·변화율은 pandas 계산값이며 CSV로 재검증 완료.

## 리포지토리 구조

```
├── data/marketing_performance.csv     입력 (280행, 8주 × 5채널 × 7일)
├── scripts/generate_insight_report.py 재현 파이프라인
├── metrics.json                       계산값 원본
├── insight_report.md                  상세 분석 리포트
├── insight_report_스크립트출력.md       스크립트 원본 출력 (재현 증거)
├── 의사결정메모.md                     결론 우선 요약 (결론 → 근거 → 반론 → 실행)
├── 기획안.md                          예산 재배분 기획안 + 기대효과 시나리오
├── decisions.md                       정제·집계 판단 로그
└── web/                               발표덱 3종 + 스크롤 리포트 2종
```
