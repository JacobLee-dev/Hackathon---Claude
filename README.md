# marketing-budget-decision

마케팅 성과 CSV 한 장을 넣으면 "다음 주 예산을 어느 채널로 옮길지"를 금액까지 답하는 리포트를 자동 생성한다.
계산은 Python(pandas)이 전담하고, 해석과 권고만 Claude가 담당한다.

```bash
python scripts/generate_insight_report.py --csv data/marketing_performance.csv
# → insight_report.md + metrics.json 재생성
```

## 결론부터

> **다음 주 메타 초과 집행분 319만원을 이메일 CRM으로 이동.** 기대 추가매출 약 1,876만원(기준 시나리오).

| 채널 | 광고비 비중 | 매출 비중 | ROI | 권고 |
|------|-----------:|---------:|----:|------|
| 이메일 | 38.5% | 46.3% | 1,042% | 유지 + 초과분 수용 |
| 네이버 | 22.5% | 17.1% | 659% | 유지·소폭 증액 |
| **메타** | **26.0%** | **7.8%** | **259%** | **일 예산 한도·이동** |
| 카카오 | 13.0% | 5.1% | 343% | W4 급락 원인 확인 |
| 오가닉 | 0% | 23.7% | 측정 불가 | 자산화 과제로 분리 |

메타 단일 채널이 유료 예산의 1/4을 쓰면서 매출은 1/13만 만든다. 예산이 효율과 반대로 배분된 상태.

전체 근거와 반론 대응은 [`의사결정메모.md`](의사결정메모.md), 상세 분석은 [`insight_report.md`](insight_report.md) 참조.

## 왜 이 리포지토리가 필요한가

DA팀은 매주 5개 채널 성과 CSV를 받지만, 정작 필요한 답인 "다음 주 예산을 어디로 옮기나"는 그 안에 정리돼 있지 않다.

- 매주 수작업 추출·계산·요약에 반나절 소요
- 엑셀 수식 계산은 오류 검증 불가
- 결과물이 수치 나열에 그쳐 의사결정으로 미연결

이 리포지토리는 리포트 자동 생성기가 아니라, **경영진이 추가 분석 없이 예산을 옮길 수 있는 결정 도구**를 목표로 한다.

## 어떻게 결론에 도달했나

질문 하나("어느 채널로 옮기나")를 가설 네 개로 쪼개 순서대로 검증했다.

1. **효율 격차가 이동을 정당화할 만큼 큰가** → 이메일 ROI가 메타의 약 4배. 확인.
2. **업계 대비 잘하는 채널인가** → 네이버는 업계 상위 25%, 이메일은 내부 1위이나 업계 평균 하회. 이메일은 증액보다 발송 최적화 우선. 부분 반증.
3. **최근 예산이 새는 구간이 있는가** → 메타 W5에서 광고비 +255.9% 대비 ROI 반감. 초과분 319만원이 그 주 예산의 35.2%. 확인.
4. **숫자를 신뢰해도 되는가** → 이메일 1건 매출이 평균의 9.6배. 미정제 시 순위가 뒤집힘. 정제 후 집계. 확인.

가설별 수치 근거는 `insight_report.md` §3~5, 정제 판단은 [`decisions.md`](decisions.md) 참조.

## 데이터 처리 규칙

결론의 신뢰성은 정제에서 나온다. 원본 280행에 다음을 적용했다.

| 처리 | 대상 | 이유 |
|------|------|------|
| 중복 제거 | 네이버 05-15 1행 | 유지 시 성과 과대 집계 |
| 결측 유지(0 대체 금지) | 3건 | 수집 누락을 0 성과로 처리 시 합계 왜곡 |
| 이상치 제외 | 이메일 04-26 (AOV 9.6배) | 포함 시 이메일 순위 왜곡. 삭제가 아닌 제외 후 명시 |
| ROI 측정 불가 처리 | 오가닉(광고비 0) | 0·무한대 표기는 오답 |

280행 → 중복 제거 279 → 이상치 제외 278로 집계. 모든 합계·ROI·변화율은 pandas 계산값이며 CSV로 재검증 완료.

## 리포지토리 구조

```
├── data/marketing_performance.csv     입력 (280행, 8주 × 5채널 × 7일)
├── scripts/generate_insight_report.py 재현 파이프라인
├── metrics.json                       계산값 원본
├── insight_report.md                  상세 분석 리포트
├── insight_report_스크립트출력.md       스크립트 원본 출력 (재현 증거)
├── 의사결정메모.md                     결론 우선 요약 (결론 → 근거 → 반론 → 실행)
├── 기획안.md                          예산 재배분 기획안 + 기대효과 시나리오
├── decisions.md                       정제·집계 판단 로그
└── web/                               발표덱 3종 + 스크롤 리포트 2종
```
