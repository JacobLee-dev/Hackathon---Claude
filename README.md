# DA 공통 제출 — 마케팅 성과 CSV를 예산 결정 리포트로 바꾸는 파이프라인

> **한 줄 요약:** 매주 반나절 걸리던 5개 채널 성과 정리를, 채널 효율 순위·급변 이슈·예산 재배분안까지 담은 결정용 리포트로 한 번의 실행에 만드는 재현 가능한 파이프라인.

- **속도** — 담당자가 반나절 걸려 수기로 계산·요약하던 리포트를 명령 1회로 대체(반나절 → 1분).
- **정확성** — 합계·ROI·변화율·이상치 탐지를 전부 Python(pandas)이 계산. 수기 계산 오류 제거.
- **의사결정** — 수치 나열에 그치지 않고, 다음 주 어느 채널을 늘리고 어디를 줄일지 금액으로 제시.
- **재현성** — 새 성과 CSV로 교체 시 동일 파이프라인이 동일 품질로 재실행. 외부 API 의존 없음.

---

## 1. 문제를 어떻게 재정의했는가

발제 원문은 "마케팅 성과 데이터로 비즈니스 인사이트 리포트를 만들 것"이었으나, 실제 해결 대상을 다음과 같이 재정의함.

> "리포트 작성"이 아니라, **경영진이 추가 분석 없이 다음 주 예산을 옮길 수 있는 결정 문서 작성.**

### 배경 (DA팀의 현실)

- 매주 5개 채널(네이버·메타·카카오·오가닉·이메일) 성과 CSV를 수작업 추출·계산·요약, 한 주기에 반나절 소요.
- 엑셀 수식·셀 참조 계산은 오류 검증 불가.
- 결과물이 수치 나열에 그쳐 "다음 주 어디에 예산을 쓰나"라는 질문에 미응답.
- 답이 나오지 않는 사이 효율 낮은 채널로 예산 지속 유입. 실제 본 데이터에도 낭비가 이미 1회 발생(메타 W5).

**성공 기준:** 리포트 분량이 아니라, 이 문서 하나로 마케팅 리드가 예산을 옮길 수 있는가.

## 2. 무엇을 만들었는가 (Basic → Standard → Challenge, 누적형)

| 레벨 | 핵심 산출물 | 실무 의미 |
|------|-----------|----------|
| **Basic** | `insight_report.md` (핵심수치·채널 ROI 순위·이슈 3·W7→W8 변화율) | 8주 5개 채널 성과를 결정 흐름 순서로 정리. 중복·결측·이상치 정제 처리 |
| **Standard** | 재현 파이프라인 `scripts/generate_insight_report.py` + `metrics.json` | 새 CSV 교체 시 동일 품질 재현. 계산은 pandas 전담, 해석은 Claude |
| **Challenge** | 예산 재배분 기획안(`기획안.md`) + 기대효과 시나리오 + 액션 오너·기한 | 채널 격차에서 재배분안 도출, 담당·기한·성공 기준·리스크까지 부착 |

## 3. 핵심 발견 — 질문 하나를 가설 네 개로 검증

분석 질문은 하나. **"다음 주 예산, 어느 채널로 옮겨야 하나."** 이 답을 위해 가설 4개를 세우고 데이터로 검증함.

| # | 가설 | 검증 결과 | 판정 |
|---|------|----------|------|
| 1 | 채널 효율 격차가 예산 이동을 정당화할 만큼 큰가 | 이메일 ROI 1,042% vs 메타 259%, 약 4배. 메타는 유료 광고비 26.0% 사용·매출 7.8% 기여 | 확인 |
| 2 | 업계 대비 잘하는 채널인가 | 네이버 ROAS 6.59로 업계 상위 25%. 이메일 10.42는 내부 1위이나 업계 평균(18.5) 하회 | 부분 반증 |
| 3 | 최근 예산이 새는 이슈가 있는가 | 메타 W4→W5 광고비 +255.9%인데 매출 +43.6%, ROI 306%→123% 반감. 초과분 319만원 = 해당 주 예산의 35.2% | 확인 |
| 4 | 숫자를 신뢰해도 되는가 | 이메일 1건 매출이 평균의 9.6배. 미정제 시 순위 왜곡 | 확인 |

**결론:** 메타 초과분 319만원을 이메일 CRM으로 이동. ROAS 격차(7.83) 기준 기대 추가매출은 보수 1,251만원 / **기준 1,876만원** / 상한 2,503만원.

## 4. 재현 가능한 파이프라인

```
CSV 입력
  → drop_duplicates()            중복 1행 제거
  → 결측 NaN 유지                 0 대체 금지, 해당 지표 합산에서만 제외
  → 이메일 04-26 이상치 제외
  → 오가닉 spend=0 → ROI 측정 불가
  → 채널별 ROI·ROAS·CTR·CVR 계산
  → 이슈 탐지 (전주 대비 ±50% 급변)
  → W7→W8 변화율 · 업계 벤치마크 판정
  → insight_report.md + metrics.json 출력
```

```bash
python scripts/generate_insight_report.py --csv data/marketing_performance.csv
```

리포트와 계산값 재생성. 새 CSV는 동일 컬럼 구조로 교체하거나 `--csv` 인수로 지정. 실제 재실행으로 수치 일치 확인 완료.

## 5. 주요 설계 결정 (방향을 바꾼 것만)

- **계산과 해석 분리.** 280행을 사람·LLM이 직접 계산하면 합계·소수점 오류 발생. pandas가 전량 계산, Claude는 해석·권고만 담당(PAL 구조).
- **결측 0 대체 금지.** 수집 누락을 0 성과로 처리 시 CTR·매출 합계 왜곡. "데이터 없음"으로 유지.
- **이상치는 삭제가 아닌 제외 후 명시.** 순위 왜곡 차단, 제외 사유 기록으로 투명성 확보.
- **오가닉은 ROI 순위 제외·별도 지표 평가.** 광고비 0원으로 수식 미성립. 매출 1.04억(전체 23.7%)·전환 5,331건은 자산으로 별도 관리.

## 7. 제약 · 금지 범위

- ROI는 발제 정의(매출 / 광고비 × 100) 사용. 순수익 ROI는 참고값 병기.
- 오가닉 ROI는 "측정 불가"로 표기. 0·무한대 미사용.
- 이상치 제외는 원본 로그 확인 전까지 잠정.
- 재배분 기대효과는 Python 계산값이며 어트리뷰션 중복 미반영 추정치. "성과 보장"·"자동 최적화" 표현 미사용.

## 리포지토리 구조

```
.
├── insight_report.md              메인 산출물 (Basic 리포트)
├── insight_report_스크립트출력.md   스크립트 원본 출력 (재현 증거)
├── 기획안.md                       예산 재배분 기획안 (Challenge)
├── 의사결정메모.md                  결론 우선 요약 (결론 → 근거 → 반론 → 실행)
├── decisions.md                   정제·집계 의사결정 로그 6건
├── metrics.json                   Python 계산값 원본
├── data/
│   └── marketing_performance.csv  입력 데이터 (280행)
├── scripts/
│   └── generate_insight_report.py 재현 파이프라인
└── web/
    ├── 발표덱-디자인1-데이터데스크.html   16:9 발표덱 (다크 터미널)
    ├── 발표덱-디자인2-에디토리얼.html     16:9 발표덱 (네이비·골드)
    ├── 발표덱-디자인3-레드브랜드.html     16:9 발표덱 (화이트·레드)
    ├── 리포트-디자인1-대시보드.html       인터랙티브 스크롤 리포트
    └── 리포트-디자인2-주최양식.html       카드형 스크롤 리포트
```

발표덱은 브라우저로 열어 좌우 방향키로 이동. PDF 저장은 인쇄(Ctrl+P) → 용지 가로 → 배경 그래픽 체크(슬라이드 1장 = 1페이지).
