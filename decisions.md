**[10:20] 계산·해석 역할 분리 = Python으로 전량 계산**
- 결정 내용: 합계·ROI·ROAS·CTR·CVR·전주변화율 전부 pandas 스크립트로 계산. Claude는 산출 수치로 해석·권고만 작성.
- 선택한 이유: 280행 수기/LLM 계산은 합계 오류·소수점 오류 위험. 재현성·검증성 확보.
- 대안: Excel 수동 계산 → 셀 참조 오류 위험으로 기각.

**[10:22] 완전 중복 1행 제거 (dedup 먼저)**
- 결정 내용: `drop_duplicates()`로 네이버광고 2026-05-15(W7) 중복 1행 제거 후 집계 (280 → 279행).
- 선택한 이유: 중복 유지 시 네이버 W7 매출·전환 부풀려짐 → 채널 비교·이슈 판단 왜곡.
- 대안: 유지 후 가중치 보정 → 불필요하게 복잡, 기각.

**[10:24] 결측 3건 = 0 대체 안 함**
- 결정 내용: impressions(카카오 04-11)·clicks(메타 04-20)·revenue(네이버 05-07) 결측을 NaN 유지, 해당 지표 합산에서만 제외(pandas 기본).
- 선택한 이유: 수집 누락을 0 성과로 보면 CTR·매출 합계 왜곡. "데이터 없음"이 정확.
- 대안: 0 대체 / 평균 대체 → 성과 왜곡·근거 부족으로 기각.

**[10:26] 이메일 2026-04-26 매출 이상치 제외**
- 결정 내용: 해당 행 매출 32,249,907원, AOV 196,646원 = 정제 후 이메일 평균 AOV(20,543원)의 9.6배. 입력 오류 의심 → 집계에서 제외(279 → 278행).
- 선택한 이유: 포함 시 이메일 ROI 과대평가 → 예산 재배분 판단 오염. 원본 로그 확인 전 보수적 제외.
- 대안: 포함 후 주석 → 순위 왜곡으로 기각. 삭제 대신 "제외+명시"로 투명성 유지.

**[10:28] 오가닉 ROI = "측정 불가" 처리**
- 결정 내용: 오가닉 spend=0 → ROI/ROAS 계산에서 `.where(spend>0)`로 제외, 순위표 별도 표기. 매출·전환·CTR·CVR로만 평가.
- 선택한 이유: 0으로 나누기 = 무한대/오류. 오가닉은 매출 1억(전체 23.7%)·고전환이라 별도 전략 지표로 다룸.
- 대안: 임의 대체값 → 근거 없음, 기각.

**[10:30] ROI 공식 = 매출/광고비×100 (문제 정의 채택)**
- 결정 내용: 채널 순위 비교는 과제 정의 `매출/광고비×100` 사용. 순수익 기준 `(매출-광고비)/광고비×100`은 참고값으로 병기.
- 선택한 이유: 채점 기준이 문제 정의식이라 그대로 따름. 순수익 관점도 함께 제시해 해석 깊이 확보.
- 대안: 순수익 ROI만 사용 → 채점 기준 불일치로 기각.

**[10:34] 이슈 3가지 선정 = 비즈니스 영향 순**
